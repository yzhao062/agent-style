revised: same budget, same register

Switch session caching from in-memory LRU to Redis with a 5-minute TTL. Sessions now survive pod restarts; per-pod memory drops from 1.2 GB to 280 MB. Trade-off: a Redis outage logs all users out, but session lookup already fails closed, so the user impact is a 401, not a 500.
