This change removes the legacy cookie-based auth middleware and makes header-based JWT via the `Authorization: Bearer` header the only supported authentication scheme. All request paths that previously accepted auth cookies now require a valid JWT in the request headers, eliminating dual-mode auth behavior and simplifying server-side enforcement. Callers must complete migration off cookie-based auth no later than **2026-06-30**. Deprecation messaging is included in the release notes, integration guidance, and operational communications so downstream teams have clear notice of the cutoff. After the deadline, requests that rely on legacy auth cookies will be treated as unauthenticated and rejected.
