## Related Work

Work on agent evaluation has already established several useful baselines, especially **BFCL**, **AgentBench**, and **tau-bench**, but these benchmarks target different constructs than the one measured here. **BFCL** emphasizes whether a model can select and format tool calls under schema constraints. **AgentBench** broadens the scope to interactive tasks across environments, making it valuable for measuring general agent competence. **tau-bench** moves closer to realistic workflows by evaluating end-to-end task completion in tool-mediated settings. The limitation, for the present benchmark, is that each mixes tool use with neighboring abilities in a different way. **BFCL** is strongest at invocation accuracy rather than sustained evidence use, **AgentBench** bundles planning, environment handling, and domain knowledge, and **tau-bench** prioritizes successful completion of realistic procedures. A benchmark designed to isolate tool-use quality needs a narrower measurement target than any one of these provides.

A second strand comes from evaluation of retrieval-augmented generation, where the central question is often whether access to external information improves final answers. That literature is closely related in spirit, but its usual unit of analysis is still answer quality, not the quality of the agent’s interaction with tools. This is precisely where the gap remains. Relative to retrieval-centered evaluation, **BFCL** does not place much weight on whether retrieved evidence was necessary, sufficient, or correctly integrated after the call. **AgentBench** often allows success through broad problem-solving ability even when retrieval behavior itself is not the decisive factor. **tau-bench** measures whether a workflow finishes correctly, but that can mask whether the agent sought information efficiently, interpreted retrieved results correctly, or recovered well from incomplete evidence. For a benchmark meant to measure tool-mediated information use, those intermediate decisions must be first-class objects of evaluation.

A third strand concerns tool use as a capability distinct from code generation. Writing code can solve tasks offline by producing a program, while tool use requires the agent to act online: decide when to query, choose parameters, interpret outputs, and update the plan from feedback. **BFCL**, **AgentBench**, and **tau-bench** all move beyond pure generation, yet none cleanly separates this interactive competence from adjacent skills. **BFCL** can reward correct call syntax without demanding deep strategic use; **AgentBench** may reward general coding or reasoning skill; **tau-bench** may reward familiarity with a fixed workflow. The new benchmark therefore treats tool use itself, rather than code production or generic task success, as the primary capability to measure.
