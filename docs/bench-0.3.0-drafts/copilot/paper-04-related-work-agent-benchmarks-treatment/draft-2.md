## Related Work

Prior benchmarks have established important baselines for LLM agents, but they target different capabilities than the one measured here. BFCL focuses on function-calling fidelity, which makes it valuable for testing whether a model can map an instruction to a valid API invocation. AgentBench broadens the picture by placing models in interactive environments that require sequential decisions across heterogeneous tasks. tau-bench moves further toward realistic workflows, emphasizing longer-horizon interaction and task completion under practical constraints. Taken together, these benchmarks show that tool use is not a single skill. At the same time, each benchmark mixes multiple sources of difficulty, so none is designed to isolate the specific measurement target of a benchmark centered on robust tool use itself: choosing when to call tools, issuing the right calls, interpreting outputs, and recovering from imperfect intermediate states under a consistent evaluation frame.

A related gap appears in evaluation of retrieval-augmented behavior. In practice, many agent tasks depend on recognizing when internal knowledge is insufficient, retrieving external information, and then grounding later actions in that evidence. BFCL, AgentBench, and tau-bench all touch adjacent behavior, because tools and environments often expose information that must be consulted before acting. However, they do not treat retrieval as a first-class measurement object. BFCL is primarily about call correctness rather than evidence acquisition. AgentBench evaluates success in broad environments, but retrieval quality is usually folded into overall task performance. tau-bench captures workflow competence, yet it likewise emphasizes end-to-end resolution over a clean account of whether the agent retrieved the right information, used it at the right time, and remained faithful to it afterward. A benchmark aimed at tool-use measurement therefore needs a tighter link between retrieval decisions and scored outcomes.

This distinction also matters because tool use should not be reduced to code generation. An agent can produce plausible code or structured output without demonstrating that it can operate tools effectively in context. BFCL comes closest to separating these abilities by testing structured invocation, but success there can still overstate real execution competence. AgentBench and tau-bench require action in environments, yet their broad task settings make it hard to disentangle tool-use skill from general reasoning, planning, or domain familiarity. A new benchmark should therefore measure tool use as its own capability: not the ability to describe actions, and not the ability to generate code, but the ability to act through tools accurately, adaptively, and verifiably.
