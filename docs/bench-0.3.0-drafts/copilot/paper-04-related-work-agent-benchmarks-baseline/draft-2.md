## Related Work

Work on evaluating LLM agents has converged on several adjacent but incomplete targets. In prior agent benchmarks, BFCL, AgentBench, and tau-bench each capture important parts of agent behavior, but none fully measure tool use as a controlled, standalone capability. BFCL foregrounds function calling fidelity and schema adherence, which is useful for testing whether a model can produce well-formed tool invocations. AgentBench broadens the scope to interactive tasks in simulated environments, emphasizing end-to-end task completion under varied constraints. tau-bench moves closer to realistic workflows by evaluating multi-step behavior in business-style settings. Taken together, these benchmarks show the field’s progression from call formatting to interactive execution, yet they also blend tool use with many other competencies, including planning, domain knowledge, and environment-specific strategy. For a benchmark intended to isolate the quality of tool use itself, that coupling makes it hard to determine whether failure comes from poor tool selection, weak execution control, or unrelated reasoning errors.

A second neighboring strand is retrieval-augmented generation evaluation. Here the central question is often whether a model can find and use external information to improve an answer. That framing overlaps with tool use because retrieval is itself a tool-mediated operation, but it typically collapses the problem into final-answer quality rather than the sequence of actions taken to obtain that answer. Relative to this goal, BFCL, AgentBench, and tau-bench still do not provide a clean measurement of retrieval-like tool interaction as an operational competence. BFCL says little about multi-step evidence gathering, AgentBench often embeds retrieval inside broader task success, and tau-bench prioritizes workflow completion over fine-grained assessment of how external information was acquired and integrated. A benchmark for tool use therefore needs to score not just outcomes, but also whether agents invoke tools appropriately, economically, and robustly.

A third strand concerns the growing tendency to treat tool use as interchangeable with code generation. BFCL, AgentBench, and tau-bench all help move evaluation beyond static text generation, but they do not fully separate acting through tools from writing code that could in principle solve the same problem offline. That distinction matters because executable code synthesis and real-time tool orchestration impose different demands: the latter requires grounding in interfaces, state, feedback, and recovery from tool errors. A new benchmark should therefore measure tool use directly, rather than treating it as a byproduct of general coding or task-solving ability.
