## Related Work

Recent evaluation work has made clear that large language model agents cannot be assessed solely through static question answering, yet existing benchmarks still leave important gaps for measuring robust tool use. In prior agent benchmarking, **BFCL**, **AgentBench**, and **tau-bench** are the most relevant touchpoints. **BFCL** is valuable for standardizing function-calling behavior and checking whether a model selects the right tool and arguments, but that framing is largely local: it emphasizes call correctness more than long-horizon interaction, state tracking, or recovery from earlier mistakes. **AgentBench** broadens the picture by placing agents in interactive environments, but its breadth also makes the measurement diffuse, since performance can depend on environment-specific knowledge or task format rather than tool-use competence itself. **tau-bench** moves closer to realistic workflows with multi-step tasks, yet it is oriented toward a particular style of operational task completion, leaving open the need for a benchmark that more cleanly isolates generalizable tool-use behavior across settings.

A second strand concerns evaluation ideas that grew out of retrieval-augmented generation. That literature highlights whether a system can identify relevant external information and ground its responses in retrieved evidence, which is clearly adjacent to tool use. However, retrieval evaluation typically treats external access as a support mechanism for answer quality, not as the central object of measurement. For a benchmark centered on tool-use, the key question is not only whether an agent can fetch useful information, but whether it can decide when retrieval is necessary, combine retrieval with other tools, manage intermediate state, and act under partial observability. Here again, **BFCL**, **AgentBench**, and **tau-bench** each touch part of the problem, but none squarely targets the boundary between information access and tool-mediated action that a dedicated benchmark should capture.

A third line of related work treats tool use as evidence of broader capability, often in contrast to code generation. This distinction matters because code generation usually evaluates offline synthesis from a fixed prompt, whereas tool use evaluates interactive competence: choosing actions, handling feedback, and coordinating with external systems. **BFCL**, **AgentBench**, and **tau-bench** all help show that success with tools is not reducible to writing code, but they do not fully separate these abilities in their measurement design. A new benchmark is therefore needed to evaluate tool use directly as an agent capability in its own right, rather than as a byproduct of API formatting, general task success, or coding ability.
