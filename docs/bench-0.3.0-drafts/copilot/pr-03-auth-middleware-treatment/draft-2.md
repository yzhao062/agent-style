This change removes the legacy cookie-based auth middleware and makes `Authorization: Bearer <JWT>` the only supported authentication scheme across the service. Requests that still rely on auth cookies will no longer be authenticated after this change lands, so all callers must migrate to header-based JWT by **2026-06-30**. The update also removes the fallback and compatibility paths that previously allowed cookie and header auth to coexist. Deprecation messaging has been added to make the cutoff explicit in logs, docs, and integration guidance for downstream consumers. This keeps authentication behavior consistent, reduces ambiguous request handling, and closes the remaining path for legacy session-style access.
