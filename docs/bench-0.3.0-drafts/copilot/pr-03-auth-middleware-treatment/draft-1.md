This change removes the legacy cookie-based auth middleware and makes header-based JWT in the `Authorization: Bearer <token>` format the only supported authentication scheme. It simplifies the request path by eliminating dual auth handling and ensures all callers use the same explicit, stateless JWT flow. All upstream callers must migrate off cookie-based auth and send header-based JWTs no later than **2026-06-30**. The deprecation message for this change is: **"Cookie-based authentication is deprecated and will be removed on 2026-06-30; migrate to Authorization header JWT immediately."** After that deadline, requests that rely on auth cookies will no longer be accepted.
