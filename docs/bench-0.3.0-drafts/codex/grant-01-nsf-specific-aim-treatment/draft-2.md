### Specific Aim 1: Adaptive Outlier Detection for Clinical Data Streams

Clinical monitoring systems need detectors that update as patients, devices, and care contexts change. The research question for this aim is: how can an outlier detector distinguish clinically meaningful anomalies from distribution shift, missingness, and routine patient-specific variation in streaming clinical data? We will answer this question by designing adaptive algorithms that combine online uncertainty estimation, drift-aware model updating, and patient-calibrated thresholds across vital signs, laboratory values, and medication-linked events.

The novelty claim is that this aim treats outlier detection as a coupled streaming inference and clinical context problem. Prior approaches often use fixed thresholds, retrospective batch training, or generic drift detectors that separate statistical change from clinical interpretation. Our approach will instead learn when to update, when to retain historical structure, and when to flag an event as anomalous under current patient context. This design will reduce false alarms from benign drift while preserving sensitivity to rare, high-risk patterns.

Task 1 will formalize the streaming detection objective. We will define loss functions that penalize delayed detection, false alarms, and unsafe adaptation. Milestones are a mathematical problem statement, benchmark protocols for at least three clinical stream types, and reproducible baseline results using fixed-threshold, batch, and online methods.

Task 2 will develop adaptive detection algorithms. We will build models that estimate uncertainty, detect covariate and concept drift, and update thresholds without using future data. Milestones are algorithms with bounded memory growth, documented update rules, and at least a 20 percent reduction in false alarm rate at matched sensitivity on held-out streams.

Task 3 will evaluate robustness and clinical plausibility. We will test performance under missing values, sensor noise, changing sampling rates, and simulated workflow delays. Milestones are stress-test reports, ablation studies identifying which components drive performance, and clinician-reviewed case summaries for representative alerts.

This aim aligns with NSF intellectual merit criteria in three ways. Scientific rigor comes from explicit objectives, streaming evaluation without temporal leakage, ablations, and uncertainty-aware error analysis. Investigator qualifications support the work because the team combines expertise in machine learning, clinical informatics, and scalable data systems. Feasibility follows from the staged plan: each task produces usable artifacts, measurable milestones, and decision points that guide later algorithm development.
