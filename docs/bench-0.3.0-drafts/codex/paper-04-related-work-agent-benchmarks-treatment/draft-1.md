## Related Work

Benchmarks for LLM agents have begun to move evaluation beyond single-turn question answering, but they often measure narrower slices of behavior than a benchmark for robust tool use requires. BFCL focuses on function calling and makes tool selection, schema matching, and argument construction explicit. This framing isolates an important interface skill, but it does not fully capture longer task horizons, state changes across calls, or recovery after a wrong or partial tool result. AgentBench evaluates agents across a broader set of interactive environments, which better reflects planning and acting under feedback. Its breadth, however, makes it harder to isolate tool-use competence from task-specific knowledge, prompt following, or environment familiarity. tau-bench is closer to realistic tool-mediated work because it emphasizes goal completion through actions, policies, and stateful interaction. Still, its intended use case leaves room for a complementary benchmark that controls tool affordances more tightly and measures where failures arise: choosing a tool, forming inputs, interpreting outputs, or deciding when the task is complete.

Retrieval-augmented generation evaluation studies a related but distinct problem. These evaluations ask whether a model can find, use, and cite information that supports an answer. That setting tests grounding, relevance, and sometimes faithfulness, all of which matter for agents that call search or database tools. Yet retrieval-centered tasks usually treat the external system as an information source rather than as an action interface. They often end once the model produces an answer, so they undermeasure tool sequencing, side effects, verification after an action, and the handling of unavailable or contradictory tool outputs. BFCL, AgentBench, and tau-bench each touch parts of this space, but none treats retrieval behavior as one component inside a controlled, multi-tool decision process whose main outcome is successful task execution.

Tool use should also be measured as a capability distinct from code generation. Code benchmarks reward producing programs that compile, pass tests, or implement a specification. Tool-use benchmarks instead ask whether a model can operate existing interfaces correctly: select an API, respect its schema, call it at the right time, update its plan from the result, and avoid unnecessary calls. A model may write correct code while still failing to use tools reliably, especially when the challenge lies in state tracking, operational judgment, or ambiguous user intent. BFCL captures the schema-facing part of this distinction, AgentBench captures broader agent behavior, and tau-bench captures stateful task completion. The new benchmark builds on these strengths while targeting tool-use reliability as the primary object of measurement.
