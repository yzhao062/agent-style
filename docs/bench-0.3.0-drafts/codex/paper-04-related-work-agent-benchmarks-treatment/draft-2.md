## Related Work

Prior agent benchmarks have made tool use observable, but they measure only part of the behavior needed by the proposed benchmark. BFCL targets function calling: it tests whether a model can select a function, fill arguments, and match a specified interface. That focus makes BFCL useful for studying API grounding, but it gives less evidence about long-horizon tool use, recovery from bad intermediate results, and stateful interaction across multiple calls. AgentBench broadens the setting by placing agents in varied tasks and environments. Its breadth, however, makes it harder to isolate which failures come from tool selection, tool execution, planning, instruction following, or domain knowledge. tau-bench moves closer to realistic service workflows, where agents must act through tools while satisfying user goals and policy constraints. Still, its task setting centers on domain-specific transactional conversations, so it does not fully separate general tool-use skill from the local structure of those workflows. The new benchmark builds on these strands, but it targets controlled measurement of tool-use competence across tasks where tools are the central interface, not incidental support.

A second related strand evaluates retrieval-augmented generation. Retrieval-augmented generation evaluation usually asks whether a system can find relevant evidence and use it to produce a grounded answer. This framing captures an important tool pattern: the model must call an external information source before answering. Yet retrieval is a narrow case of tool use. The action space is usually search, read, and synthesize, while many agent settings require choosing among heterogeneous tools, passing structured inputs, composing outputs, and deciding when more tool interaction is useful. Retrieval evaluation also tends to judge final answer faithfulness or evidence use. The proposed benchmark instead measures the process by which an agent turns a user goal into a sequence of tool calls and observations. For that purpose, retrieval-style scoring is insufficient because a correct final answer can hide brittle tool choice, unnecessary calls, or lucky recovery from earlier mistakes.

Tool use should also be distinguished from code generation. Code generation measures whether a model can write programs, often with syntax, algorithmic reasoning, and library knowledge as central skills. Tool use measures a different interface: the model must understand available operations, choose among them, supply valid arguments, interpret returned observations, and adapt its plan. BFCL touches this distinction through function calling, while AgentBench and tau-bench test broader agent behavior. The proposed benchmark treats tool use as a first-class capability rather than a proxy for programming skill or general task success.
