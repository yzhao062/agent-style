This PR switches the session cache from the in-process LRU implementation to Redis so session state can be shared consistently across app instances. The rollout plan is to deploy Redis configuration behind a feature flag, enable it first in staging, then ramp production traffic gradually while monitoring cache hit rate, latency, error rate, and fallback behavior. If Redis errors or latency regressions exceed the rollback threshold, we will disable the flag and return to the existing in-process cache path. The single biggest trade-off is adding an external infrastructure dependency in exchange for cross-instance consistency and more predictable behavior during horizontal scaling.
