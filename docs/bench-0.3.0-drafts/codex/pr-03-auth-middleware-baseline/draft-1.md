This PR removes the legacy cookie-based auth middleware and makes `Authorization: Bearer <JWT>` the only supported authentication path. Cookie parsing, fallback session lookup, and related middleware wiring are deleted so requests without a valid header JWT now fail consistently. Callers must migrate to header-based JWT authentication by June 30, 2026. The deprecation message for remaining cookie users is: `Cookie-based authentication is deprecated and will be removed; send a Bearer JWT in the Authorization header instead.` This keeps the auth surface smaller, removes ambiguous precedence between cookies and headers, and aligns all clients on a single token validation flow.
