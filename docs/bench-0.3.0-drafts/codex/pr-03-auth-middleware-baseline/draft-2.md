This change removes the legacy cookie-based authentication middleware and makes header-based JWT validation the only supported authentication path. Requests must now provide a valid `Authorization: Bearer <token>` header, and cookie credentials are no longer inspected or accepted. Callers that still rely on cookie-based auth must migrate to header-based JWT before the migration deadline of June 30, 2026. The deprecation messaging is updated to state that cookie-based auth has been removed and to direct integrators to use bearer tokens instead. Removing the legacy path simplifies authentication behavior, eliminates an outdated fallback, and makes unsupported callers fail consistently.
