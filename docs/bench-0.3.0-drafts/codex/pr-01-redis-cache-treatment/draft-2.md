This PR replaces the in-process LRU session cache with Redis-backed storage so sessions survive process restarts and can be shared across app instances. The implementation routes session reads and writes through Redis, keeps the existing cache interface, and uses configured TTLs to preserve current expiration behavior. Rollout plan: deploy Redis configuration to staging, enable the Redis cache for one canary instance, compare hit rate and latency against the LRU baseline, then expand to all instances. The single biggest trade-off is that session access now depends on Redis availability and network latency instead of local memory.
