This change removes the legacy cookie-based authentication middleware and makes header-based JWT authentication the only supported scheme. Callers must migrate to `Authorization: Bearer <jwt>` by the migration deadline of June 30, 2026. Requests that rely on auth cookies will no longer be accepted after that deadline. During the deprecation window, callers should treat the existing warning message, `Cookie-based authentication is deprecated; migrate to header-based JWT before June 30, 2026`, as the required migration notice. This simplifies the auth path, removes legacy middleware behavior, and aligns all clients on one JWT transport mechanism.
