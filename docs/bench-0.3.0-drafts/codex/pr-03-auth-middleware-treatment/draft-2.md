This PR removes the legacy cookie-based auth middleware and makes header-based JWT authentication the only supported scheme. Callers must send a valid JWT in the `Authorization: Bearer <token>` header by June 30, 2026. Requests that still rely on auth cookies will no longer authenticate after that deadline. Until the cutoff, deprecation messaging should tell callers: "Cookie-based authentication is deprecated and will be removed on June 30, 2026; migrate to Authorization: Bearer JWT." This change simplifies auth handling, removes the dual-path middleware behavior, and aligns all clients on one explicit token transport.
