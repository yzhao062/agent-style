## Related Work

Prior agent benchmarks have established important foundations for measuring language models in interactive settings, but they only partially capture the tool-use behavior targeted by our benchmark. BFCL emphasizes function calling and structured API invocation, making it useful for testing whether a model can select and format calls correctly. However, its focus on call correctness leaves less room for measuring longer-horizon behavior such as recovering from tool errors, coordinating multiple tools, or deciding when not to call a tool. AgentBench broadens the scope by evaluating agents across diverse environments, but that breadth can make it difficult to isolate tool-use competence from task-specific reasoning, environment familiarity, or prompt sensitivity. tau-bench moves closer to realistic service workflows and tests agents in conversational, policy-constrained settings, yet it remains centered on particular transactional scenarios. Our benchmark is intended to measure tool-use as a general agent capability: planning with tools, grounding intermediate decisions in observations, handling state, and completing tasks under realistic interaction constraints.

Retrieval-augmented generation evaluation forms a second relevant strand because many agent tasks depend on acquiring external information before answering or acting. Existing RAG evaluations commonly emphasize retrieval quality, answer faithfulness, citation accuracy, or end-to-end response correctness. These dimensions are necessary but insufficient for evaluating tool-using agents. A model may retrieve the right information but fail to integrate it into a multi-step workflow, call retrieval at the wrong time, over-retrieve when the answer is already available, or ignore contradictory tool outputs. Conversely, an agent may need to combine retrieval with other tools, update its plan after observing results, and preserve task state across turns. BFCL, AgentBench, and tau-bench each touch parts of this space, but none treats retrieval as one tool among many in a controlled measurement of tool orchestration.

Finally, tool-use should be distinguished from code generation as a capability measure. Code generation benchmarks often reward producing a static artifact that passes tests, while tool-use requires acting through external interfaces whose outputs shape subsequent decisions. Even when tools are APIs or functions, the central challenge is not merely writing executable code or syntactically valid calls; it is selecting the right operation, supplying grounded arguments, interpreting observations, and adapting. BFCL captures aspects of this distinction through function-calling evaluation, while AgentBench and tau-bench expose agents to interactive consequences. Still, the intended measurement of our benchmark is narrower and more diagnostic: it evaluates whether models can use tools reliably as part of agentic problem solving, rather than treating tool calls as a proxy for programming skill or general task success.
