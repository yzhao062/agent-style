## Related Work

Benchmarks for LLM agents increasingly evaluate models in settings that require acting rather than merely answering. BFCL focuses attention on function calling, emphasizing whether a model can select the right callable interface and produce valid arguments. AgentBench broadens the frame to interactive agent tasks in which a model must make progress through sequential decisions. tau-bench adds an emphasis on realistic task completion under tool-mediated interactions. These benchmarks are important precursors for measuring agents as systems, but they leave open a narrower measurement problem targeted here: isolating tool-use behavior with enough control to distinguish planning failures, schema failures, state-tracking errors, and recovery behavior. BFCL is strong on call formatting but less diagnostic of multi-step tool policies; AgentBench is broad but can entangle tool use with domain knowledge and environment-specific strategy; tau-bench stresses realism but can make it difficult to attribute failures to particular tool-use capabilities.

Retrieval-augmented generation evaluation provides a second relevant strand because retrieval is one of the most common external-tool patterns for LLM systems. Such evaluations typically ask whether a model can use retrieved evidence to answer correctly, remain faithful to sources, or avoid unsupported claims. This is related to tool use, but it measures a different interface between model and environment. A retrieval setting often hides many operational choices: when to query, how to reformulate a query, how to combine multiple tool results, and how to proceed when retrieved evidence is incomplete or conflicting. BFCL, AgentBench, and tau-bench each touch parts of this broader interaction space, yet none treats retrieval-style evidence access as one instance of a unified tool-use measurement framework. For the proposed benchmark, the central issue is not only whether external information improves the final answer, but whether the agent uses tools appropriately over the full trajectory.

Tool use also differs from code generation as a capability measure. Code benchmarks evaluate whether a model can produce an executable artifact, often in a single response or bounded repair loop. Tool-use benchmarks instead test whether a model can act through constrained APIs, respect schemas, interpret observations, and adapt future actions to previous tool outputs. BFCL captures part of this distinction through function-call validity, while AgentBench and tau-bench evaluate richer interaction patterns. However, existing coverage still leaves gaps for measuring tool use independently from programming skill. A model may write correct code yet fail to choose the right tool, pass valid arguments, preserve state across calls, or recover from tool errors. The proposed benchmark therefore treats tool use as a first-class agent capability rather than as a proxy for coding ability or general task success.
