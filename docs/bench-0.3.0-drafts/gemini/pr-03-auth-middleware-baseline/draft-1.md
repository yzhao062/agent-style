This pull request officially removes the legacy cookie-based authentication middleware, finalizing our transition to header-based JWT as the only supported scheme. Effective immediately, any requests relying on session cookies will be rejected with a 401 Unauthorized status and a "Legacy authentication is no longer supported" error message. All service callers must migrate their authentication headers to the Authorization: Bearer <token> format by the final sunset deadline of July 1, 2026. We have updated our deprecation warnings to notify users that the old middleware is non-functional and that JWT issuance is now required. This consolidation reduces our security attack surface, simplifies our internal middleware logic, and ensures a more consistent identity model for all API consumers.
