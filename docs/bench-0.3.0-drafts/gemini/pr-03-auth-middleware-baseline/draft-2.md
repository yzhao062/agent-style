This pull request removes the legacy cookie-based authentication middleware to establish header-based JWT as the exclusive supported authentication scheme. This consolidation simplifies our security architecture and ensures consistent authentication patterns across all service endpoints. All external callers must migrate to the `Authorization: Bearer` header format before the mandatory deadline of July 1, 2026. To facilitate this, we have added a deprecation header stating: "Warning: Cookie authentication is deprecated and will be disabled on July 1, 2026; please migrate to JWT headers." After the deadline expires, the system will strictly enforce JWT validation and reject all cookie-dependent session requests with a 401 Unauthorized status.
