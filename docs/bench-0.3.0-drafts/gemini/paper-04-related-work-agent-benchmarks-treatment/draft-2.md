## Related Work

Research into Large Language Model (LLM) agents currently follows three primary trajectories: general agentic benchmarking, retrieval-augmented generation (RAG) evaluation, and code generation assessment. While these areas provide foundational insights, they do not isolate the specific mechanics of tool-use within complex, stateful environments.

Current agent benchmarks provide essential metrics for task completion and instruction following. AgentBench serves as a comprehensive framework across diverse environments, but its breadth often masks the granular errors associated with multi-step tool interactions. While AgentBench measures overall success, it lacks the focus required to diagnose specific failures in API parameterization or sequence logic. BFCL addresses this by evaluating function-calling precision in single-turn interactions. However, BFCL focuses on the selection of the correct function rather than the sustained reasoning needed to resolve dependencies between multiple tools. Similarly, tau-bench evaluates agent performance in domain-specific tasks such as retail or travel. tau-bench introduces realistic user-agent dialogues but prioritizes conversational alignment over the technical robustness required for system-level tool manipulation. These three benchmarks collectively highlight a need for a measurement tool that stresses long-horizon planning and error recovery.

Evaluation of RAG systems constitutes a separate strand of inquiry. These frameworks assess the ability of a model to retrieve relevant information and generate faithful responses. However, RAG metrics treat the model as a consumer of static knowledge. This approach ignores the active role an agent plays when it must modify its environment or execute state-changing actions. RAG evaluation fails to capture the complexity of managing tool-mediated state, where the output of one action dictates the parameters of the next.

Finally, tool-use is frequently categorized as a sub-type of code generation. Conventional code generation benchmarks measure syntactic accuracy and algorithmic correctness. Tool-use requires distinct capabilities, including the interpretation of external API contracts and the handling of side effects in dynamic systems. Conflating these capabilities obscures the specific challenges of grounding agentic actions in a persistent environment. By separating tool-mediated interaction from raw code synthesis, this benchmark provides a dedicated measure for the interactive and operational integrity of LLM agents.
