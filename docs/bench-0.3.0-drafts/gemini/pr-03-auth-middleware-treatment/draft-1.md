This pull request removes the legacy cookie-based authentication middleware to establish header-based JWT as the sole supported authorization scheme. API consumers attempting to authenticate via cookies will receive a `401 Unauthorized` response stating that cookie-based sessions are no longer supported. All callers must update their implementations to use the `Authorization: Bearer <token>` header by the June 1, 2026 migration deadline. The removal of session-state tracking simplifies the service architecture and eliminates cross-site request forgery vulnerabilities. Developers should verify their local environments and integration tests to ensure they comply with these mandatory security updates.
