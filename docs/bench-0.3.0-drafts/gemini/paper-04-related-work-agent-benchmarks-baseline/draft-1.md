## Related Work

The evaluation of LLM agents has transitioned from simple text completion to multi-step problem solving in interactive environments. Early efforts like AgentBench established a multi-domain framework for measuring agentic capabilities across tasks such as database manipulation and web browsing. However, AgentBench often prioritizes overall task success over the granular analysis of tool-use mechanics, complicating the isolation of specific API interaction failures. Similarly, the Berkeley Function Calling Leaderboard (BFCL) provides a rigorous assessment of syntactically correct function calls but frequently lacks the long-horizon reasoning and stateful persistence required for autonomous agents in dynamic settings. More recently, tau-bench has introduced realistic, user-facing scenarios requiring agents to coordinate multiple actions. While tau-bench captures the complexity of real-world workflows, it does not specifically stress-test boundary cases like handling ambiguous API documentation or recovering from unexpected tool errors, which are central to the benchmark proposed here.

Evaluation in retrieval-augmented generation (RAG) primarily focuses on document retrieval and grounded generation. However, these paradigms fail to capture the interactive complexity of tool-use where agents must actively influence external systems. While AgentBench includes some information retrieval tasks, it does not evaluate the precision of tool-integrated retrieval as a primary mechanic. Furthermore, BFCL focuses on the syntax of the call rather than the relevance of retrieved data within a multi-step RAG pipeline, and tau-bench emphasizes conversational flow over the technical nuances of active data fetching. Consequently, frameworks like AgentBench and BFCL do not adequately measure the transition from passive knowledge retrieval to active tool-driven exploration.

Finally, tool-use must be recognized as a capability distinct from general code generation. Code generation benchmarks measure a model's ability to synthesize logic, whereas tool-use focuses on its capacity to orchestrate existing APIs. Existing evaluations often conflate these skills. For instance, while AgentBench and tau-bench incorporate programming environments, they do not sufficiently isolate protocol adherence from general algorithmic reasoning. Unlike BFCL, which evaluates function calling in isolation, our benchmark stresses protocol consistency across complex sequences. By distinguishing these capabilities, we show how AgentBench and tau-bench can obscure tool-specific failures by merging them with broader coding errors.
