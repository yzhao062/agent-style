## Related Work

The landscape of autonomous agent evaluation has been shaped by several foundational frameworks, yet a significant gap remains in measuring nuanced tool-use capabilities in dynamic environments. Early efforts such as AgentBench provided a multi-dimensional platform for assessing agents across diverse tasks, including database manipulation and web browsing. However, AgentBench often prioritizes general-purpose reasoning over the specific, fine-grained mechanics of complex external API interaction. BFCL (Berkeley Function Calling Leaderboard) was developed to specifically target the precision of function calling. While BFCL offers a rigorous assessment of syntax and parameter alignment, it remains largely atomized, focusing on single-turn tool invocations rather than the long-horizon, stateful interactions required in real-world deployments. Similarly, tau-bench introduced more realistic, user-centric environments to test agent reliability in complex domains. Despite these advancements, tau-bench and its predecessors frequently conflate domain-specific success with generalizable tool-use proficiency, often failing to isolate the core capability of selecting and sequencing tools in highly ambiguous contexts.

The evaluation of retrieval-augmented generation (RAG) represents a second critical strand of related research. Traditional RAG metrics emphasize the accuracy of document retrieval and the faithfulness of the subsequent generation. However, these evaluations typically treat the retrieval mechanism as a static component of the pipeline. They do not sufficiently account for scenarios where the agent must treat the retrieval engine as a proactive tool that requires strategy, query refinement, and iterative interaction. While benchmarks like AgentBench and tau-bench include elements of information retrieval, they often lack the granular telemetry needed to distinguish between a failure in linguistic synthesis and a failure in the strategic deployment of retrieval tools.

Finally, we distinguish tool-use from code generation as a distinct capability measure. While code generation benchmarks evaluate the ability to produce executable logic, tool-use requires an agent to interact with existing, often opaque, external interfaces. BFCL addresses the interface aspect but lacks the complex logic of full-scale programming. Conversely, code generation tasks often allow for "closed-world" execution where the agent has total control. Tool-use, as measured by our benchmark, requires navigating "open-world" constraints that are not captured by the static code evaluation in AgentBench. Our benchmark fills this void by isolating tool-use as a unique strategic skill set.
