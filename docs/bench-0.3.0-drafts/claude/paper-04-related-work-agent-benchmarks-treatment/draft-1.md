## Related Work

Three lines of prior work frame the gap this benchmark targets: agent evaluation suites, retrieval-augmented generation evaluation, and efforts to disentangle tool use from general code generation ability.

Existing agent benchmarks each probe a different slice of tool-use competence. BFCL scores single- and multi-turn function calling against curated schemas and rewards syntactic fidelity to API specifications; the evaluation is largely closed-form and environment-free, so a model that picks correct arguments against a known schema scores highly without demonstrating recovery when a tool returns an unexpected payload. AgentBench broadens the surface to interactive environments such as operating system shells, databases, and web pages, and reports aggregate success on long-horizon tasks. Its per-environment heterogeneity makes it difficult to isolate tool-selection skill from planning, memory, and environment-specific priors. tau-bench introduces scripted user simulators that enforce airline and retail policies and rewards strict policy compliance rather than raw task completion; its two domains and templated user turns leave open whether a model generalizes to tools outside its training distribution. None of the three isolates the question our benchmark asks: whether a model can choose, compose, and repair tool calls when the schema itself is partially unreliable.

Retrieval-augmented generation evaluation addresses a related but distinct target. Standard RAG evaluations grade answer faithfulness and citation accuracy given a passage set, treating retrieval as a passive enrichment step rather than an action the model authors. They therefore do not measure whether a model reasons about which tool to invoke, when to invoke it, or how to reconcile conflicting results across calls. Tool use is not passage assembly.

A third thread treats tool use as a special case of code generation, reporting pass@k on function-synthesis tasks that happen to emit JSON arguments. Code-generation metrics reward surface correctness of a single emission and miss the control-flow core of agentic tool use: deciding under uncertainty whether to call at all, which of several overlapping tools to prefer, and how to interpret error signals. Measuring tool-use capability on code-generation yardsticks conflates two skills that dissociate in practice.
