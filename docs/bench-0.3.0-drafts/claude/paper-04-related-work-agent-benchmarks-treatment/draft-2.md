## Related Work

Agent benchmarks have grown in parallel with tool-calling LLMs, yet each established suite answers a different question than the one this benchmark poses. BFCL evaluates function-calling accuracy at the API boundary, scoring whether a model emits a correctly typed invocation for a single prompt. AgentBench extends the frame to multi-turn interaction across eight environments, measuring task completion under shell, database, and web settings. tau-bench narrows the lens to customer-service dialogues, pairing a policy document with a user simulator to test whether the agent respects domain rules across a conversation. Each contributes a valuable signal, and each stops short of the measurement this paper targets. BFCL treats the tool invocation as terminal and therefore cannot observe how downstream tool output reshapes an agent's plan. AgentBench rewards task success but aggregates across environments whose failure modes have little in common, which makes diagnosis of the underlying capability difficult. tau-bench depends on a simulator whose distribution is narrow by design, and its scoring conflates policy adherence with tool selection.

Retrieval-augmented generation evaluation occupies an adjacent space but measures a different thing. Standard RAG suites score answer quality conditioned on retrieved passages, treating retrieval as a fixed upstream component. An agent with tools faces a harder problem: the agent chooses when to call a retriever, how to phrase the query, and how to reconcile retrieved content with prior tool output. A benchmark that reports only end-to-end answer accuracy hides the chain of decisions that produced it, and thus gives weak signal about tool-use competence per se.

A third strand treats tool use as a sub-capability of code generation, evaluating agents through programmatic tasks whose solutions happen to invoke APIs. This conflation misleads. Code generation rewards syntactic fluency in a programming language, while tool use rewards accurate selection among heterogeneous, noisily documented interfaces, recovery from tool-side errors, and calibrated abstention when no tool fits. The three named benchmarks each address fragments of these dimensions; the present work targets the intersection that none covers directly.
