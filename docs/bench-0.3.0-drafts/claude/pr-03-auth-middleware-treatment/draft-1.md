This change removes the legacy cookie-based authentication middleware, leaving header-based JWT (`Authorization: Bearer <token>`) as the only supported scheme. Callers must migrate by 2026-06-01; after that date, requests presenting session cookies instead of a bearer token receive a 401 Unauthorized. Since 2026-01-15, the cookie middleware has added a `Deprecation` response header and emitted an `auth.cookie.deprecated` log line on every authenticated request, with the migration guide URL in the response `Link` header. We notified downstream service owners in #api-consumers on 2026-03-01, and the deprecation dashboard now shows cookie-authenticated traffic below 0.2% of total authenticated requests. Post-merge, the `/auth/legacy/*` routes return 410 Gone and echo the migration guide URL in the response body.
