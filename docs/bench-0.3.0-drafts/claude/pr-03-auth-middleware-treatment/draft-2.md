This pull request removes the legacy cookie-based authentication middleware and establishes header-based JWT (`Authorization: Bearer <token>`) as the only supported scheme. Callers must complete migration by 2026-06-01; after that date, any request presenting a session cookie in place of a bearer token receives a 401 Unauthorized response. Since 2026-01-15, the cookie middleware has emitted a `Deprecation` response header, an `auth.cookie.deprecated` log line on each authenticated request, and a `Link` header pointing to the migration guide. Service owners received direct notice in #api-consumers on 2026-03-01, and the deprecation dashboard now reports cookie-authenticated traffic at 0.2% of total authenticated volume. Once merged, the `/auth/legacy/*` routes return 410 Gone and include the migration guide URL in the response body.
