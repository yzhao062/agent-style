## Rate limiting

The API enforces two independent token buckets: a per-IP bucket covering anonymous traffic at 60 requests per minute, and a per-token bucket covering authenticated traffic at rates tied to plan tier (1,000 per minute for free accounts, 10,000 per minute for paid accounts). Both buckets refill continuously and share a single Redis cluster for consistency across edge nodes. Every response, whether success or rejection, carries X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers so clients pace themselves without guessing. Requests that exhaust a bucket return HTTP 429 with a Retry-After header and a machine-readable error code identifying which bucket tripped. Developers who believe a limit fired incorrectly, for example when a shared NAT collapses many users onto one IP or a token is misclassified as free tier, may file an appeal at api-support@example.com with the request ID from the 429 response; support reviews within two business days and issues a scoped override when the case is valid.
