Removes the legacy cookie-based authentication middleware and makes header-based JWT (`Authorization: Bearer <token>`) the only supported scheme across all API endpoints. The cookie path was formally deprecated on 2026-01-15, with every request returning a `Deprecation` response header and a `Sunset: Fri, 15 May 2026 00:00:00 GMT` header pointing callers to the migration guide, plus structured log warnings on the server side. All internal services and documented third-party integrations have already migrated, and telemetry shows cookie-auth traffic is now below 0.2% of total requests over the last 30 days. Any remaining callers must cut over to bearer-token headers by the hard deadline of **2026-05-15**; after that date, requests missing a valid `Authorization` header will be rejected with `401 Unauthorized`. This change also deletes the associated cookie-parsing helpers, session-token persistence code, and related tests, closing out the compliance-driven rewrite tracked in the auth-middleware epic.
