This PR removes the legacy cookie-based authentication middleware and makes header-based JWT (`Authorization: Bearer <token>`) the only supported auth scheme across all API endpoints. The cookie path was marked deprecated on 2026-01-15, and since then every cookie-authenticated response has carried a `Deprecation: true` header, a `Sunset: Fri, 15 May 2026 00:00:00 GMT` header, and a `Link` pointer to the migration guide, with matching server-side log warnings tagged for caller identification. All first-party services and documented integration partners have already cut over, and cookie-auth traffic is now under 0.2% of total requests over the trailing 30 days. Any remaining callers must migrate to bearer-token headers by the hard deadline of **2026-05-15**; after that cutoff, requests without a valid `Authorization` header will be rejected with `401 Unauthorized`. Alongside the middleware, this change deletes the cookie-parsing helpers, session-token persistence layer, and associated tests, closing out the compliance-driven auth rewrite.
