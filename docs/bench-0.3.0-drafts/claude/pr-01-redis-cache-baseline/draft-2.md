Switches the session cache backend from an in-process LRU to Redis so sessions persist across process restarts and are shared across all app instances. Rollout is gated behind the `session_cache_backend` flag: we will enable Redis for 10% of traffic, monitor hit rate and p99 latency for 24 hours, then ramp to 50% and 100% over the following two days, with a single flag flip to revert. The biggest trade-off is that every session lookup now crosses the network, adding roughly 1–2 ms of p50 latency compared to the in-memory hit path. Connection pooling and pipelining are configured, and a circuit breaker falls back to a short-TTL local cache if Redis becomes unavailable.
