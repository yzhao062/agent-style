## Related Work

Our benchmark draws on three lines of inquiry: agent evaluation, retrieval-augmented generation assessment, and the isolation of tool use as a capability distinct from program synthesis.

Several suites measure the broad capabilities of LLM-based agents. BFCL (the Berkeley Function Calling Leaderboard) focuses on function-calling accuracy across single and parallel invocations, stressing schema adherence and argument synthesis. AgentBench assembles a heterogeneous set of environments—operating systems, databases, web browsing, and games—to stress-test decision-making over long horizons. tau-bench centers on conversational tool use in customer-service domains, pairing a user simulator with tool-bearing agents and scoring trajectories against a reference policy. Each has shaped the field, but none isolates the capability we target: BFCL's stateless, one-shot calls underweight the coupling between successive tool outputs; AgentBench bundles planning, memory, and domain knowledge into each task-level score, so regressions in tool use are hard to attribute; and tau-bench's reliance on a simulated interlocutor injects variance from the user model itself, obscuring mechanical tool behavior.

A parallel strand evaluates retrieval-augmented generation via answer faithfulness, passage recall, and end-to-end question-answering. These protocols presume a fixed retriever–reader interface and grade whether the generator grounds its output in retrieved passages. They map poorly onto agent tool use, where the model actively chooses when and how to query, must compose results across calls, and is judged on side effects rather than textual faithfulness. BFCL's single-turn structure, AgentBench's task-level scoring, and tau-bench's dialogue-centric evaluation each sidestep this question; retrieval becomes an invisible component of a larger score rather than a measured behavior in its own right.

A third strand equates tool use with program synthesis, measuring whether a model can emit executable snippets that invoke APIs. While informative about syntactic fluency, such measurements conflate code correctness with strategic selection, sequencing, and error recovery—the behaviors that dominate real agent workloads. None of BFCL, AgentBench, or tau-bench cleanly separates these axes: BFCL grades surface-level call shape, AgentBench rewards task completion without decomposing it, and tau-bench entangles dialogue competence with tool choice. Our benchmark targets the unfilled middle.
