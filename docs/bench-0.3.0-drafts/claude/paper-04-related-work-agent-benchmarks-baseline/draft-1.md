## Related Work

Our benchmark sits at the intersection of three lines of evaluation research, each of which motivates its design while leaving gaps that it is intended to fill.

**Agent and tool-use benchmarks.** A first strand evaluates language model agents on their ability to invoke external functions and complete multi-step tasks. BFCL established a now-standard protocol for measuring function-calling fidelity, scoring whether a model emits syntactically valid calls whose arguments match a ground-truth signature. AgentBench broadened the lens to interactive environments, stressing long-horizon decision making across operating systems, databases, and games. tau-bench added a crucial conversational dimension, grading an agent's ability to satisfy customer-service policies through back-and-forth dialogue with a simulated user. Valuable as these are, each makes a design choice that limits what can be concluded about tool use as a capability. BFCL largely treats function calling as a single-turn mapping problem, so it cannot observe how errors compound across a plan. AgentBench mixes tool selection with environment dynamics, making it hard to attribute failures to tool reasoning rather than to planning or state tracking. tau-bench couples tool competence to policy adherence inside a narrow domain, so strong performance does not generalize cleanly to arbitrary APIs. Our benchmark isolates the tool-use signal while still requiring multi-step composition.

**Retrieval-augmented generation evaluation.** A second strand treats external knowledge access as a retrieval problem, asking whether a model can fetch and integrate relevant passages before generating an answer. Such evaluations reward faithful grounding but assume the retriever's interface is fixed and read-only. They therefore say little about an agent's ability to choose among many heterogeneous tools, to read structured responses, or to chain stateful calls whose outputs feed subsequent inputs—exactly the behaviors our tasks target.

**Tool use as a capability distinct from code generation.** A third strand, often adjacent to program synthesis, measures whether a model can write executable code that happens to call libraries. Collapsing tool use into code generation conflates two skills: producing correct syntax and deciding which capability to invoke under uncertainty. None of BFCL, AgentBench, or tau-bench isolates the latter at scale with controlled tool inventories. Our benchmark addresses this by varying tool availability, ambiguity, and redundancy while holding surface syntax constant.
