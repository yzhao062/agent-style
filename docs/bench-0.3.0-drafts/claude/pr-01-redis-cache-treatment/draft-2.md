Switches the session cache backend from an in-process LRU to Redis so that sessions persist across application restarts and stay consistent across horizontally scaled instances. Rollout proceeds behind the `session_cache_backend` flag, starting at 1% of traffic and ramping through 10%, 50%, and 100% over four days while we monitor p99 latency and cache hit rate. If metrics regress, flipping the flag back to `lru` reverts behavior without a redeploy. The largest trade-off is that every session read now crosses the network, moving lookup latency from sub-millisecond in-process access into the low-millisecond range in exchange for cross-instance consistency and restart durability.
